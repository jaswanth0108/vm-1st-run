How to Change Admin Username & Password Safely in the Database
==============================================================

Important: You CANNOT simply type your new password directly into the database. The backend uses `bcrypt` encryption to protect passwords. If you put a plain-text password in the database, the login API will crash/fail because it won't be able to decrypt it.

Follow these 2 steps to change it without any errors.

STEP 1: Generate a Bcrypt Hash for your new password
----------------------------------------------------
You must encrypt your new password into a bcrypt hash before saving it. 

1. Open your terminal and navigate to your backend folder:
   cd "c:\Users\Jaswanth N.V.S\OneDrive\Desktop\1st run\Backend\college-exam-api"

2. Run the following command. Replace "YourNewPassword123" with your actual desired password:
   node -e "require('bcryptjs').hash('YourNewPassword123', 10).then(console.log)"

3. It will print out a long string of random characters, looking something like:
   $2a$10$abcdefghijklmnopqrstuvwxyz123...
   (Copy this exact string. This is your new password hash!)


STEP 2: Update the Database in pgAdmin 4
----------------------------------------
Now you will tell the database what the new username and new hashed password are.

1. Open pgAdmin 4 and connect to your live database server.
2. Expand your servers -> Databases -> Right-click your database and select "Query Tool".
3. Paste the following SQL query into the tool:

   UPDATE users 
   SET 
       username = 'my_new_admin_username', 
       password_hash = 'PASTE_YOUR_COPIED_BCRYPT_HASH_HERE' 
   WHERE role = 'admin';

4. Change 'my_new_admin_username' to your desired username.
5. Change 'PASTE_YOUR_COPIED_BCRYPT_HASH_HERE' to the hash you copied in Step 1.
6. Click the "Execute" button (or press F5) to run the query.

You're done! You can now go to your Admin Portal and securely log in using your new username and your new plain-text password.
